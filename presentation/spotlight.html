Intent Stabilization

Selective Resampling Before Clarification

We want assistants to avoid bluffing about user intent. Our proposal is to stabilize interpretation first, and only ask a clarification question if structured resampling still cannot lock onto the user's meaning.

Problem Statement

Why current assistants still fail

02 / 09

Failure mode 1

The request is underspecified, but the assistant answers immediately under an unspoken assumption.

Failure mode 2

The assistant clarifies too early and turns every ambiguous request into an unnecessary back-and-forth.

Failure mode 3

The assistant says it is “confident” without separating answer confidence from intent confidence.

Motivation

The repo’s legacy benchmark already showed that direct answering performs badly on ambiguous tasks. The pivoted project asks a sharper question: can we spend compute to stabilize intent and reduce unnecessary clarification?

Operational target

  • maximize final task success
  • preserve action appropriateness
  • control clarification cost
  • control extra model-call budget

Formalization

Computational framing

04 / 09

Latent-intent decision problem

Observed request: x
Latent user intent: z
Action set: a ∈ {answer, hedge, clarify, narrow, alternatives}

Policy objective:
maximize E[ Success(a, z) - λ_q · 1{clarify} - λ_c · calls ]
Stage confidences: r_j = agreement(stage_j samples)
Overall intent confidence:
c = Calibrate( 0.5 · min_j r_j + 0.5 · mean_j r_j )

Weak stage:
j* = argmin_j r_j

Interpretation

  • This is a cost-sensitive sequential decision problem.
  • The system treats user intent as a hidden state to infer.
  • Clarification is a fallback action, not the default response to every ambiguous input.
  • Selective resampling is a compute allocation policy: spend more calls only where uncertainty is concentrated.
Audience takeaway: this is a principled inference-and-control problem Speaker cue: 45–50s
Method

resample_clarify in one slide

05 / 09

1. Sample

Sample ambiguity judgments, intent models, and strategy decisions with moderate temperature.

2. Aggregate

Compute agreement, dominant interpretation, weak points, and calibrated intent confidence.

3. Repair

Select the weakest stage and resample only that stage, up to a small repair budget.

Fallback logic

  • if intent is stable: answer / narrow / alternatives
  • if intent stays unstable: ask a targeted clarification
  • if still unresolved later: defer or abstain when necessary
