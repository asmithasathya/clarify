# Training And Post-Training

The training path is intentionally modular.

## Teacher Rollouts

Teacher rollouts are generated by running `resample_clarify` over `InfoQuest train`. Each rollout stores:

- stage traces
- stabilized intent outputs
- final action/response
- optional rejected baseline comparison for preference data

Main script:

```bash
./.venv/bin/python scripts/export_teacher_rollouts.py \
  --config configs/report_baselines.yaml \
  --dataset infoquest \
  --split train \
  --output-dir outputs/teacher_rollouts
```

Outputs:

- `teacher_rollouts.jsonl`
- `sft_corpus.jsonl`
- `preference_corpus.jsonl`

## LoRA SFT

The SFT path trains a smaller student on the teacher's structured stage outputs. This lets the student reuse the same modular runtime.

```bash
./.venv/bin/python scripts/run_student_sft.py \
  --config configs/report_baselines.yaml \
  --train-corpus outputs/teacher_rollouts/sft_corpus.jsonl \
  --output-dir outputs/student_sft
```

The training helper currently uses the installed `tinker` SDK directly.

Artifacts:

- `student_sft_manifest.json`
- `student_sft_history.jsonl`

The manifest contains `checkpoint_path`, which can be extracted with:

```bash
./.venv/bin/python scripts/read_manifest_value.py \
  --path outputs/student_sft/student_sft_manifest.json \
  --key checkpoint_path
```

## Student Evaluation

```bash
./.venv/bin/python scripts/run_student_eval.py \
  --config configs/report_baselines.yaml \
  --sft-checkpoint "$(./.venv/bin/python scripts/read_manifest_value.py --path outputs/student_sft/student_sft_manifest.json --key checkpoint_path)" \
  --output-dir outputs/student_eval
```

The student comparison matrix evaluates:

- base `Qwen3-4B + targeted_clarify`
- base `Qwen3-4B + resample_clarify`
- distilled `Qwen3-4B-SFT + targeted_clarify`
- distilled `Qwen3-4B-SFT + resample_clarify`

Optional DPO variants can be added later.

## Optional DPO

The repo currently prepares the DPO stage rather than forcing it into the paper-critical path.

```bash
./.venv/bin/python scripts/run_student_dpo.py \
  --config configs/report_baselines.yaml \
  --preference-corpus outputs/teacher_rollouts/preference_corpus.jsonl \
  --sft-checkpoint "$(./.venv/bin/python scripts/read_manifest_value.py --path outputs/student_sft/student_sft_manifest.json --key checkpoint_path)" \
  --output-dir outputs/student_dpo \
  --prepare-only
```

This writes a manifest that can be handed off to a cookbook-based DPO workflow when available.

## Intended Gates

- Only train the student after `resample_clarify` beats `targeted_clarify` on the teacher dev/test criteria.
- Only prepare or run DPO if the SFT student closes a meaningful fraction of the teacher-wrapper gain.
- Keep ClarifyBench held out for evaluation; do not train on it.
