\begin{abstract}
We study intent stabilization for ambiguous user requests. The repository implements an inference-time wrapper that samples multiple structured interpretations of a request, measures disagreement across ambiguity and intent stages, performs a repair pass when the intent state is unstable, and asks a clarification question only when uncertainty remains high. The codebase now supports the full research workflow around that method: frozen train/dev/test data splits, baseline and ablation matrices, teacher rollout export, modular student distillation, optional DPO preparation, and a living manuscript whose tables are generated directly from experiment artifacts. The first completed development-sweep win improves InfoQuest-dev task-success rate from \texttt{0.24} for \texttt{targeted\_clarify} to \texttt{0.29} for \texttt{resample\_clarify} with matched \texttt{1.00} appropriate-action rate, but the first fresh \texttt{InfoQuest test / qwen3\_30b} gate underperforms both \texttt{targeted\_clarify} (\texttt{0.26}) and \texttt{generic\_clarify} (\texttt{0.275}) at \texttt{0.22}. Partial ablations already indicate why: the initial selective weak-point controller over-spent compute and still clarified almost always, while a simpler full-repair pass looks stronger. This manuscript therefore keeps all unfinished reruns, ablations, audits, repeatability checks, and post-training artifacts as explicit placeholders rather than silently substituting stale or incompatible numbers.
\end{abstract}
